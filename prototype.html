Development tool

We’re choosing Android Studio because the app we’re creating is for the Android platform. One of the primary reasons for using Android Studio and not Xamarin is because Android Studio builds exclusively to Android, which means there are more options available such as ConstraintLayouts, Fragments, etc.

Design Guidelines

The following piece of text references to chapters from the book. The numbers before each explanation is the chapter that the guideline is taken from.

15: We added an animation to opening the menu when sliding from left to right. This way, the user will know what his/her action will do from the moment the menu starts to appear at the left side of the screen. When opening a chat the chatbox comes up and goes down with an animation. When the user gets a notification, it comes up from the bottom as an animation.
16: We use the same theme for our buttons and layout in the application. This way the user will know the function of certain buttons by just looking at it.
18: When the user performs an action like installing a game remotely, the user gets a small popup at the bottom so it doesn’t interrupt the user.
19: Opening the menu can be quickly undone by swiping right to left.
20: The only real mode we have it the menu mode, accessible from every page. It can be opened by a button on the top left or by swiping left to right. The menu goes away when you go to a new page or swipe right to left.
21: We have no preferences.
22: Our hierarchy is very small, there is a menu with a few main items, these were determined by card sorting tests.
23: Every action has a response time of 0.1 seconds or less.
